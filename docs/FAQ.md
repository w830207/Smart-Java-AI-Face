## 安装和配置

### Q1: 需要什么环境才能运行本项目？

**A:** 
- **JDK**: 21 或更高版本
- **Maven**: 3.6 或更高版本
- **浏览器**: 支持摄像头的现代浏览器（Chrome、Edge、Firefox等）
- **系统**: Windows、Linux、macOS 均可
- **内存**: 建议 4GB 以上

### Q2: 模型文件在哪里下载？

**A:**
- **百度网盘**: https://pan.baidu.com/s/10l22x5fRz_gwLr8EAHa1Jg?pwd=1234
- **提取码**: 1234
- **文件大小**: 
  - retinaface.pt (105MB)
  - elasticface.pt (250MB)

### Q3: 模型文件应该放在哪里？

**A:** 有两种推荐方式：

**方式一（推荐）**: 放在项目上一级目录
```
Face_recognition/
├── smart_java_ai/          # 模型目录
│   ├── retinaface.pt
│   └── elasticface.pt
└── smart_java_ai_face/     # 项目代码
```
配置为：`public static String MODAL_PATH = "../smart_java_ai/";`

**方式二**: 放在任意目录，然后修改配置
```java
// 例如：D:/models/smart_java_ai/
public static String MODAL_PATH = "D:/models/smart_java_ai/";
```

### Q4: 如何修改模型路径配置？

**A:** 
1. 打开文件：`src/main/java/com/codeying/utils/FaceRecUtils.java`
2. 找到第 58 行左右的 `MODAL_PATH` 配置
3. 修改为你的模型文件目录路径
4. **注意**: 路径末尾必须有斜杠 `/` 或 `\\`

### Q5: 能否不下载模型文件？

**A:** 不能。模型文件是人脸检测和识别的核心，必须下载。由于文件较大（355MB），无法放在 GitHub 仓库中，需要单独下载。

## 启动问题

### Q6: 启动时提示找不到模型文件怎么办？

**A:** 

**错误信息**:
```
❌ 错误：找不到模型文件 retinaface.pt
```

**解决步骤**:
1. 确认已下载模型文件
2. 检查文件路径是否正确
3. 检查 `MODAL_PATH` 配置是否正确
4. 使用绝对路径避免相对路径问题

### Q7: 启动时提示端口被占用怎么办？

**A:**

**方案一**: 停止占用 8080 端口的程序
```bash
# Windows
netstat -ano | findstr :8080
taskkill /PID <进程ID> /F

# Linux/Mac
lsof -i :8080
kill -9 <进程ID>
```

**方案二**: 修改应用端口

编辑 `src/main/resources/application.properties`:
```properties
server.port=8081  # 改为其他未占用的端口
```

### Q8: 首次启动为什么这么慢？

**A:** 

首次启动慢的原因：
1. **下载 PyTorch 运行库** (~300MB): 自动下载，只需一次
2. **加载人脸识别模型**: 加载到内存需要时间
3. **初始化数据库**: 创建 SQLite 数据库

**正常情况**:
- 首次启动: 30-60秒
- 后续启动: 5-10秒（模型已缓存）

### Q9: 启动后如何验证是否成功？

**A:**

查看控制台日志，成功标志：
```
✅ 模型文件检查通过
...
Tomcat started on port(s): 8080 (http)
Started App in 2.666 seconds
```

然后访问: http://localhost:8080/face/manage

## 功能使用

### Q10: 浏览器无法访问摄像头怎么办？

**A:**

**解决步骤**:
1. **检查权限**: 浏览器地址栏左侧 → 点击锁图标 → 网站设置 → 摄像头 → 允许
2. **刷新页面**: 授权后刷新页面
3. **使用 localhost**: 确保访问 `localhost:8080` 而不是 IP 地址
4. **尝试其他浏览器**: Chrome、Edge 支持最好

**注意**: 某些浏览器在非 HTTPS 下限制摄像头访问，但 `localhost` 通常被视为安全。

### Q11: 访问管理中心显示 404 怎么办？

**A:**

**常见原因**:
1. **应用未启动**: 检查控制台，确认已看到 "Started App" 日志
2. **端口错误**: 确认访问 `http://localhost:8080/face/manage`
3. **直接打开 HTML**: 不要在 IDE 中右键 HTML 文件打开，必须通过浏览器访问 8080 端口

**正确方式**:
1. 先运行 `App.java` 启动应用
2. 再用浏览器访问 `localhost:8080/face/manage`

### Q12: 人脸注册失败怎么办？

**A:**

**常见原因及解决**:

1. **未检测到人脸**
   - 确保光线充足
   - 正对摄像头
   - 距离适中（30-50cm）

2. **图片质量差**
   - 使用清晰的图片
   - 避免模糊、遮挡

3. **后端错误**
   - 查看控制台日志
   - 检查模型是否正确加载

### Q13: 人脸识别不准确怎么办？

**A:**

**优化建议**:

1. **注册时**:
   - 光线充足
   - 正面拍摄
   - 表情自然
   - 多角度注册（可注册多张）

2. **识别时**:
   - 与注册环境相似
   - 正对摄像头
   - 表情相似

3. **调整阈值**:

编辑 `FaceRecUtils.java`:
```java
// 在 queryFace 方法中
faceSearchParams.setThreshold(0.8f);  // 默认0.8，可调整
```

- **阈值越高**: 越严格，误识别少，但可能漏检
- **阈值越低**: 越宽松，漏检少，但可能误识别

### Q14: 如何删除人脸数据？

**A:**

**方式一**: 通过界面删除
1. 访问管理中心
2. 点击"人脸管理"选项卡
3. 点击"刷新列表"
4. 找到要删除的人脸，点击"删除"

**方式二**: 手动删除
1. 删除图片: `face-imgs/用户ID_时间戳.jpg`
2. 编辑索引: `smart_java_ai/face_index.json` 删除对应记录

## 性能优化

### Q15: 如何提升识别速度？

**A:**

1. **使用 GPU 加速**:
   - 修改配置使用 GPU 版本 PyTorch
   - 需要 NVIDIA 显卡

2. **降低图片分辨率**:
   - 修改摄像头分辨率设置

3. **增加内存**:
   ```bash
   java -Xmx2048m -jar target/proj-boot-1.0-SNAPSHOT.jar
   ```

4. **使用高速模型**:
   - 参考 SmartJavaAI 文档切换到高速模型

### Q16: 内存不足怎么办？

**A:**

**错误信息**:
```
OutOfMemoryError: Java heap space
```

**解决方案**:

**方式一**: IDE 中配置
- Run → Edit Configurations
- VM options: `-Xmx2048m`

**方式二**: 命令行启动
```bash
java -Xmx2048m -jar target/proj-boot-1.0-SNAPSHOT.jar
```

**方式三**: 修改 Maven 配置
```bash
export MAVEN_OPTS="-Xmx2048m"
mvn spring-boot:run
```

## 错误处理

### Q17: JSON 解析错误怎么办？

**A:**

**错误信息**:
```
SyntaxError: Unexpected token '<'
```

**原因**: 前端收到 HTML 而不是 JSON，通常是后端接口返回了错误页面

**解决**:
1. 检查后端是否启动成功
2. 打开浏览器 F12 → Network 标签
3. 查看接口返回的实际内容和状态码
4. 根据状态码定位问题：
   - 404: URL 路径错误
   - 500: 后端程序错误，查看日志

### Q18: 数据库相关错误怎么办？

**A:**

SmartJavaAI SDK 使用 SQLite 存储人脸特征。

**常见问题**:
- 数据库文件损坏: 删除 `smart_java_ai/` 下的 `.db` 文件重新生成
- 权限问题: 确保目录有写权限

### Q19: 模型加载失败怎么办？

**A:**

**错误信息**:
```
Failed to load model...
```

**解决步骤**:
1. **检查文件完整性**: 确认文件大小正确
2. **重新下载**: 模型文件可能损坏
3. **检查权限**: 确保有读取权限
4. **查看日志**: 查看详细错误信息

## 其他问题

### Q20: 可以部署到服务器吗？

**A:** 

可以！部署步骤：

1. **打包应用**:
   ```bash
   mvn clean package -DskipTests
   ```

2. **上传服务器**:
   - 上传 JAR 文件
   - 上传模型文件

3. **后台运行**:
   ```bash
   nohup java -jar proj-boot-1.0-SNAPSHOT.jar > app.log 2>&1 &
   ```

4. **配置 Nginx** (可选):
   - 反向代理
   - 配置 HTTPS

### Q21: 支持多个人同时识别吗？

**A:** 

当前版本主要支持单人识别。如需多人同时识别，可以：
- 修改前端代码，实现多人脸框检测
- 调用SDK的多人脸识别接口
- 参考 SmartJavaAI 官方文档

### Q22: 数据安全如何保证？

**A:**

**隐私保护措施**:
1. **本地存储**: 所有数据存储在本地，不上传云端
2. **人脸特征**: 存储的是特征向量，不是原始图片
3. **图片加密**: 可自行实现图片加密存储
4. **访问控制**: 建议添加登录认证

**注意**:
- `.gitignore` 已配置忽略人脸图片
- 不要将人脸数据上传到公开仓库
